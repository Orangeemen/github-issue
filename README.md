## 🚀 快速开始

### 提交新镜像申请
1. 🔍 **搜索现有 issues**：确认您的申请未重复提交
2. 📋 **阅读收录规则**：[POLICY.md](./POLICY.md)
3. ✍️ **新建 Issue**：选择模板 `🪞 新镜像申请`
4. ✅ **填写完整信息**：帮助维护者高效评估

👉 [🔗 点击此处提交新申请](../../issues/new?template=0-mirror-request.yml)

## 📋 收录规则摘要

我们欢迎符合以下条件的镜像申请：

✅ **必须满足**
- 使用 OSI 认可的开源许可证，或明确允许镜像分发
- 上游源稳定可用，有明确的同步方法文档
- 预估存储占用合理（常规项目 < 500GB）
- 有真实用户需求或使用场景

❌ **不予收录**
- 闭源/商业软件（无官方授权）
- 内容频繁变更且无版本控制
- 涉及版权争议或法律风险
- 纯个人项目且无广泛社区需求

📖 **完整规则**：[🔗 POLICY.md](./POLICY.md)
